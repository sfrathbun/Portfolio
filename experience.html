Data Security Compliance



Data Security Integration and Remediation | February 2024 - Present

As the Director of Product Delivery, I led a comprehensive data discovery project focused on integrating and leveraging Azure Information Protection (AIP) file scan results. I worked directly with our customer to plan, build, and oversee the development of an integration that pulls in detailed file-level data and identifies files requiring remediation by tagging sensitive information types detected by AIP, including Credit Card numbers and PII data. Building on this foundation, I designed and implemented a remediation program that generates automated notifications and tickets for end users to clean up files on their corporate endpoints. Additionally, I established a structured support process, ensuring our support team could guide end users and property teams in effectively remediating files within scope. I was actively involved throughout the project lifecycle, from initial integration development to implementing the remediation workflow and defining the support processes that guarantee successful adoption and compliance.

Integrations

  • Microsoft Azure Information Protection - Pulling data directly from on-prem database servers.
  • C# - Data parsing and email notification program.
  • ServiceNow API integration - Created tickets to track and manage the remediation workflow.
  • Endpoint Security Compliance Tool - Generated reports and dashboards showing remediation progress

My contributions to this project

  • Conducted daily stand-up meetings with the customer to collaboratively develop the remediation plan and oversee the project lifecycle from initiation to completion.
  • Developed PowerPoint and Visio presentations to effectively communicate project progress and milestones to leadership. Additionally, these materials visually documented the full integration workflow, helping stakeholders quickly understand the process and technical approach.
  • Collaborated with the customer to define the support process, develop comprehensive documentation and FAQs, and train the support team to effectively guide end users through issue remediation.
  • During a three-week pilot, we identified over 11,000 files containing credit card numbers. Following the pilot, all files were successfully remediated, with our support team actively assisting end users to ensure proper remediation and compliance.

Web Server Logging



Splunk Web Server Logging and Compliance | November 2023 - Present

I led the design and implementation of an enterprise web server logging compliance initiative to ensure all hosts running web servers centrally log both access and error events to Splunk. The solution automatically discovers active web servers across Windows and Linux environments, validates that logging is functioning as expected, and measures compliance by generating and verifying real log activity. Confirmed log data is ingested into a centralized database, enabling accurate reporting and visibility into logging compliance across the enterprise.

This effort required close collaboration with multiple application and infrastructure teams to support a wide range of server types and web technologies, including both property and above-property systems. I also partnered with the support organization to establish operational processes for follow-up and remediation, ensuring application owners address gaps and maintain continuous compliance with logging requirements.

Integrations

  • Shell scripting - Custom Splunk Technical Add-On (TA) to identify running web servers and their active listening ports. Local web server probing to intentionally generate access and error logs for validation
  • C# - Splunk log verification to confirm successful ingestion of generated log events
  • Compliance data ingestion into an internal database for reporting and metrics
  • PowerShell scripting - Windows host discovery scripts to detect active web servers locally.
  • Remote IIS probing to validate IIS access and error logs are correctly forwarded to Splunk.
  • Multi-platform support for both Linux and Windows web servers across diverse environments
  • Operational support workflows to track non-compliant systems and coordinate remediation with application owners.

Data Platform Reliability and Performance



I lead ongoing efforts to ensure the reliability, scalability, and performance of our enterprise data platform, which supports over 100 API integrations and programs which ingests over 100 million rows of data daily. The platform sustains billions of read and write operations each day, requiring deliberate design, continuous monitoring, and proactive optimization to maintain consistent performance at scale.

To address throughput and stability challenges, I re-architected data ingestion workflows by implementing controlled batch and file chunking strategies, significantly reducing I/O contention and improving load predictability during high-volume data processing. I also tuned MySQL database configurations and storage-level I/O parameters, along with other critical global settings, to better align the database engine with real-world workload demands. In parallel, I established standards and review practices to ensure application queries are efficient, non-blocking, and free of excessive locking behaviors that could degrade concurrent operations. Collectively, these efforts have strengthened platform resiliency, improved responsiveness, and positioned the data platform to scale reliably as usage and data volumes continue to grow.
