ANGULAR 9, Spring Boot, and MySQL

A connecting hub for people and Churches to build a better city... together.

An event posting app for Churches and organizations to better their city | Team Project -June 2020

For our final project at Bethel Tech, our team built Hope City; an application designed to link people, Churches, and organizations together, uniting with one purpose... to build a better city. This application allows a user to sign up and log into their account. A person can request to post events on the website. The events must be geared towards helping the city and the people in it. The user can then post their events to that people can sign up and volunteer for these events. The idea is to get people connected and involved with their city.

The front end was built with Angular 9 using Material Design Bootstrap. The website includes login, password hashing, JWT, and cookies. The back end was built with a Java Spring Boot rest API so that users can post safely to our MySQL database. Check out our Hope City project HERE

My contributions to this project

I spent a portion of this project working on the events page. This consisted of creating an HTTP request to the back end. I created all of the methods for events such as the get, create, update, and delete. I implemented a modal window so that when a user clicks an event they see the event info in the modal. Below is a list of my other contributions to this project.

  • Form validation for sign up and create and editing events.
  • Most all of the UI using Material Design Bootstrap
  • Implemented Google Maps API on our footer
  • Events Page
  • Prayer Page

REACT APP

Neighboring Application for the COVID-19 Global Church Hackathon.

COVID-19 Global Church Hackathon Team Project -April 2020

In response to the Coronavirus outbreak, Faith Tech put on a Hackathon challenge. With 10 challenges to choose from and one week until completion, our team of 12 developers created a website that would serve our neighbors with their basic needs. A user creates a profile and can request food and household items they are lacking. The local Church would then step in to provide for the needs of the community. There is also a way to give in order to help supply the needs of others as well.

The front end was built with REACT JS using react-bootstrap. The website includes login, password hashing, JWT, user and admin profile views. The back end was built with EXPRESS JS so that users can post their needs to a database. All available items are stored in the database as well as the user and admin profile information. I am grateful to be part of this dynamic team.

My contributions to this project

As a part of the front end team, the majority of my contributions consisted of creating the footer, the give page, the profile page, and some of the home page. I built the home page slider and created the images for it, as well as most of the other images throughout the site. Team standup meetings were held daily to regroup and share our progress. Because this was a one-week Hackathon challenge, we worked intensely as a team, communicated remotely through zoom, and used GitHub to source control all of our pushes to the master branch. It was truly an amazing effort by the entire team given the short amount of time we had.

EXPRESS JS Social Media App

Express JS, Handlebars JS, and MySQL database

Social Media App using MySQL database

This is a fully functioning social media app that was built for a Bethel Tech school project. It allows users to sign up, login, and create posts. This app uses salt and password hashing for extra security measures. JSON web tokens are used to allow the user to navigate through the app without being logged out. The back end was built with EXPRESS JS so that users can post to the database. All posts are stored in the database as well as the user and admin profile information. Certain users may also have admin privileges, giving them access to private information, also allowing them to delete posts and users from the database.

Angular 8 with JSON Mock Server
